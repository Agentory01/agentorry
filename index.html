🛡️ Security May 2026 · 4 min read

How we secured Agentorry from the ground up

Building a marketplace where people upload and download code is one of the most security-sensitive things you can do on the internet. Here is every layer we built.

A
Agentorry Team
Engineering · @agentorry01

When you build a marketplace where people upload ZIP files and other users download and run them — you are sitting on one of the most dangerous attack surfaces on the web. We knew this from day one and built security into every layer.

Layer 1 — Database Security

Every table in our database has Row Level Security (RLS) enabled. This means even if someone gets hold of your API key, they can only access the data that belongs to them. No user can read, write, or delete another user's data.

We also built audit logs that track every sensitive action — every product uploaded, every price changed, every product published or unpublished. If something suspicious happens, we can trace it back to the exact timestamp and user.

"Security is not a feature you add at the end. It is the foundation everything else is built on."

Layer 2 — Input Sanitization

Every piece of text that enters our database goes through sanitization triggers. HTML tags are stripped. JavaScript injection attempts are blocked. SQL injection patterns are removed. This happens at the database level — not just the frontend — so even direct API calls cannot inject malicious content.

Layer 3 — ZIP Validation

Every uploaded ZIP goes through our validation engine before it can be published. Here is what we check:

🔍
AI Framework Detection
We scan for LangChain, OpenAI, Anthropic, Gemini, AutoGPT, and 10+ other frameworks. If a file claims to be an AI agent, it must actually contain AI agent code.
⛔
Malware Detection
We scan for known malware keywords — rootkits, ransomware, reverse shells, keyloggers. Any match results in immediate rejection and account review.
💣
ZIP Bomb Protection
ZIP bombs are archives that expand to hundreds of gigabytes when extracted. We detect and block these before they can cause damage.
🚫
Executable Blocking
.exe, .bat, .ps1, .cmd files are blocked unless accompanied by strong AI agent signals. No reason for a legitimate AI agent to include Windows executables.

Layer 4 — Rate Limiting

Spam accounts cannot flood the marketplace. We limit uploads to 20 per day and 10 per hour. Reviews are capped at 10 per day. New accounts must wait before uploading. These limits stop bots and bad actors without affecting legitimate users.

Layer 5 — Fraud Prevention

You cannot buy your own product. You cannot review your own product. You cannot wishlist your own product. Purchase records are immutable — nobody can modify or delete them. Product ownership cannot be transferred. Every one of these rules is enforced at the database level, not just the UI.

Layer 6 — Private File Storage

ZIP files are stored in a private bucket. Raw storage URLs are never exposed. Downloads are served through signed URLs that expire after 60 seconds. You cannot download a paid product without a verified purchase record.

Security is never done

We are constantly improving. If you find a vulnerability, please contact us immediately.
